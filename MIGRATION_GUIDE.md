# 🚀 Руководство по миграции с Vanilla JS на Next.js

## Что изменилось?

### Архитектура
| Аспект | Было (Vanilla JS) | Стало (Next.js) |
|--------|-------------------|-----------------|
| **Рендеринг** | Client-Side Only | SSR + ISR + CSR |
| **Роутинг** | HTML файлы | File-based routing |
| **Состояние** | localStorage | React Query + Zustand |
| **API** | `/api/data.js` | `/app/api/data/route.ts` |
| **Авторизация** | localStorage (небезопасно) | NextAuth JWT (httpOnly cookies) |
| **Типизация** | Нет | TypeScript |
| **Стили** | Vanilla CSS | Tailwind CSS |

### Структура URL (автоматические редиректы)
```
article.html?id=123    → /news/123
team.html?id=5         → /teams/5
player.html?user=nick  → /players/nick
index.html             → /
teams.html             → /teams
...
```

## 🔄 Процесс перехода

### Шаг 1: Запуск нового сайта
```bash
cd efl-next
npm install
cp .env.example .env.local
# Заполните .env.local данными MongoDB
npm run dev
```

### Шаг 2: Проверка данных
1. Откройте `http://localhost:3000`
2. Проверьте, что данные загружаются из MongoDB
3. Проверьте авторизацию (старые пароли работают)

### Шаг 3: Тестирование

```bash
npm run build

```

### Шаг 5: Переключение DNS
1. Убедитесь, что новый сайт работает корректно
2. Обновите DNS записи домена
3. Старый сайт оставьте как fallback на 7 дней

## ⚠️ Важные замечания

### Безопасность
- **Пароли:** В старой версии пароли хранились в plaintext. NextAuth поддерживает их для совместимости, но новые пароли должны хэшироваться через bcrypt
- **Сессии:** Теперь используются JWT в httpOnly cookies (безопаснее localStorage)

### Производительность
- **ISR:** Главная страница кэшируется на 60 секунд
- **CSR:** Страницы с фильтрами используют client-side рендеринг
- **React Query:** Автоматическое кэширование запросов

### SEO
- **Metadata API:** Каждая страница имеет правильные мета-теги
- **Open Graph:** Настроены превью для соцсетей
- **Sitemap:** Генерируется автоматически

## 🐛 Известные проблемы и решения

### Проблема: Старые пользователи не могут войти
**Решение:** NextAuth поддерживает plaintext пароли для миграции. После первого входа пользователь может сменить пароль, и он будет захеширован.

### Проблема: Данные не загружаются
**Решение:** Проверьте `MONGODB_URI` в `.env.local` и маппинг коллекций в `lib/db.ts`

### Проблема: 404 на старых URL
**Решение:** Все редиректы настроены в `next.config.mjs`. Проверьте конфигурацию.

- FCP: ~0.8s
- LCP: ~1.5s
- Bundle size: ~95KB (code splitting)
